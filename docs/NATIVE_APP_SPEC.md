# ネイティブiOSアプリ 仕様書（端末内オフラインAI）

「記憶を持つAI」を **iPhone本体だけ・完全オフライン・無制限** で動かすネイティブ版の設計仕様。
Web版（Cloudflare + Gemini）に対し、こちらは **端末内で小型LLMを直接実行**する。

> 本書のコードは [`../ios/`](../ios/) にサニタイズ済みで置いています（個人情報・APIキー・本番URLは含みません）。

---

## 1. ねらい / 背景
- Web版は高品質だが、**通信・無料枠・（ローカル推論時は）母艦Mac**に依存する。
- 「電波が無くても・無制限に・自分のメモを参照して話したい」を満たすには **端末内推論** が要る。
- 検証の結果、ブラウザ(WebLLM/WebGPU)はiOS Safariのメモリ制限で **0.5〜1Bが限界**＝品質不足。
  → **ネイティブ化**すれば 3B が実用的に動く（メモリ上限を引き上げられる）。

## 2. 技術スタック
| 層 | 採用 | 理由 |
|---|---|---|
| UI | SwiftUI | iOS標準・最短で構築 |
| 推論 | **MLX-Swift**（`mlx-swift-examples` の MLXLLM / MLXLMCommon） | Apple Silicon最適化。`ChatSession` で履歴管理が簡単 |
| モデル | **Qwen2.5-3B-Instruct（4bit, 約1.8GB）** | 3B級で日本語が比較的強い。iPhone(6GB)で動く |
| 記憶 | 端末内 JSON ＋ メモリ内キーワード検索 | 小規模Vault(<1MB)なのでDB不要・即時 |
| 記憶取得 | クラウドの `/api/vault/export`（要認証）から同期 | オフライン参照のため端末に保存 |

## 3. 構成ファイル（[`../ios/`](../ios/)）
| ファイル | 役割 |
|---|---|
| `AinasApp.swift` | アプリのエントリポイント |
| `ContentView.swift` | チャットUI＋「記憶の同期」シート |
| `AinasEngine.swift` | MLXでモデル読込・推論・人格(system)・会話履歴（`ChatSession`） |
| `ChatMessage.swift` | メッセージのデータ型 |

## 4. データフロー

```
[書く]  Obsidian(メモ)
          │ ① 母艦が起動中に クラウドDB(D1) へ同期
          ▼
        クラウド D1（記憶ミラー）  ──②同期エンドポイント /api/vault/export（要認証）
          │ ③ アプリの「🔄同期」(オンライン時)でpull
          ▼
[使う]  iPhone内に保存（JSON）
          │ ④ 質問時に端末内でキーワード検索 → 上位数件を抽出
          ▼
        MLX(Qwen2.5-3B) が「私の記憶」として受け取り、端末内で推論 → 回答
```

## 5. オフライン動作（重要）
| 機能 | オフライン(機内モード) |
|---|---|
| 会話・推論 | ✅ 可（端末内で完結） |
| 同期済みの記憶を参照 | ✅ 可（端末に保存済み） |
| 新しい記憶の同期 | ❌ 不可（クラウドDBへ通信が要る） |

設計思想：**「読む・思い出す・話す＝オフライン完結」「新しく覚える＝オンライン時に同期」**。

## 6. 記憶検索アルゴリズム（`VaultStore.search`）
1. クエリから検索語を抽出：ASCII語・カタカナ語・**漢字連＋2-gram**・ひらがな3字以上・カナ→英シノニム。
2. 各チャンクを走査し、**パス/タイトル一致は本文一致の×3**で重み付けスコア。
3. 上位 N 件（既定4件）の本文を返し、プロンプトへ注入。

→ 日本語は分かち書きが難しいため、この素朴な方式でも「タイトル一致重視＋表記揺れ吸収」で実用的な再現率になる。

## 7. メモリと品質のトレードオフ
- iPhone 15(6GB) では **3B-4bit** が安定上限。落ちる場合は `AinasEngine.modelId` を `Qwen2.5-1.5B-Instruct-4bit` に下げる（品質↓・安定↑）。
- 端末内3Bは Web版(Gemini/70B級)より品質は落ちる。**「無制限・オフライン」を品質より優先**する設計。

## 8. セキュリティ / プライバシー
- 同期は**ユーザーが入力するパスワード**で認証（コード・バイナリに秘密を埋め込まない）。
- メモ本文は端末内にのみ保存（クラウド常時接続不要）。
- 自作アプリのため iOS の **Developer Mode** が必要（自分専用配布）。
